Extraction rules:
- For line items like "16,470.00 lb @ $291.00/ton", parse quantity=16470, quantity_unit=LBS, rate=291.00, rate_unit=TON.
- If the description includes a ration code (e.g., W1J or W13), keep it in description.
- Financial Summary rows should be captured as-is with their Current/Previous/Total values.
- If a section is absent, return an empty list for that section.
- Preserve leading zeros in lot_number.

CRITICAL - Amount accuracy:
- Read each digit in the Total column very carefully. Common OCR errors: 9↔5, 1↔7, 8↔6, 3↔8.
- The total_period_charges MUST equal the sum of all line_items[].total values.
- Look for "Total Charges" or "Net Amount" in the Current column of Financial Summary for the period total.
- If you see a "Current Balance" row, that is the total_amount_due.
- Capture ALL line items - do not skip any rows in the Period Charges table.
